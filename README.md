# 후원 룰렛

치지직·숲 방송에서 들어오는 후원 메시지를 룰렛 선택지로 등록하고, 후원 금액만큼 표를 자동으로 더해 주는 웹앱입니다.

- 방송인은 웹앱에서 **플랫폼, 채널 ID, SSAPI API 키, 1표당 금액**만 입력하면 됩니다.
- 룰렛 항목·득표 수·후원 기록은 사용 중인 컴퓨터의 브라우저에 저장됩니다.
- 치지직·숲 공식 API는 사용하지 않습니다. 채널명과 대표 이미지는 원하면 직접 입력합니다.

---

## 1. 사용 전에 준비할 것

| 준비물 | 설명 |
| --- | --- |
| Node.js | 프로그램을 내 PC에서 실행하거나 GitHub Pages에 올릴 때 필요합니다. Node.js LTS 버전을 설치하세요. |
| SSAPI API 키 | 실제 후원 메시지와 금액을 받기 위해 필요합니다. |
| 내 채널 ID | 치지직 또는 숲에서 어떤 방송의 후원을 받을지 정하는 값입니다. |

> 이 프로젝트는 SSAPI Socket API를 이미 사용하도록 만들어져 있습니다. 별도로 `websocket-client`, Python 파일, 서버 프로그램을 실행할 필요가 없습니다.

---

## 2. 처음 한 번만 하는 SSAPI 설정

후원 데이터는 SSAPI를 통해 받아옵니다. 따라서 **API 키 발급**과 **내 채널 등록**만 먼저 해 주세요.

### 2-1. SSAPI API 키 발급

1. SSAPI 대시보드에 로그인합니다.
2. API 키를 발급받습니다.
3. 발급받은 키는 다른 사람에게 공유하지 말고 보관합니다.

### 2-2. 내 방송 채널 등록

SSAPI 대시보드의 스트리머 등록 메뉴에서 본인 방송을 등록합니다.

- 플랫폼: `치지직` 또는 `숲`
- 스트리머 ID: 본인 채널 ID
- 등록한 채널과 웹앱에 입력할 채널은 **같은 플랫폼·같은 채널 ID**여야 합니다.

치지직 주소 예시:

```text
https://chzzk.naver.com/live/75cbf189b3bb8f9f687d2aca0d0a382b
```

위 주소에서 채널 ID는 맨 뒤의 아래 값입니다.

```text
75cbf189b3bb8f9f687d2aca0d0a382b
```

> SSAPI에 채널 등록이 되어 있지 않으면 웹앱이 연결되어도 해당 방송의 후원이 들어오지 않습니다.

## 3. 내 PC에서 실행하기

### 3-1. Node.js 설치 확인

PowerShell을 열고 아래 두 명령을 실행합니다.

```powershell
node -v
npm -v
```

각각 버전 숫자가 나오면 준비가 된 것입니다.

`node` 또는 `npm`을 찾을 수 없다고 나오면 Node.js LTS를 설치한 뒤 PowerShell을 새로 열어 주세요.

### 3-2. 프로젝트 폴더로 이동

압축을 푼 폴더 위치에 맞춰 이동합니다.

```powershell
cd C:\Users\thdal\Downloads\stream-roulette\stream-roulette
```

### 3-3. 필요한 라이브러리 설치

아래 명령을 **먼저 하나만** 실행합니다.

```powershell
npm install
```

처음 설치할 때는 인터넷 상태에 따라 몇 분 걸릴 수 있습니다. 설치가 끝나서 PowerShell 프롬프트가 다시 나타난 뒤에만 다음 단계로 넘어가세요.

```text
(base) PS C:\Users\thdal\Downloads\stream-roulette\stream-roulette>
```

### 3-4. 웹앱 실행

`npm install`이 끝난 다음 아래 명령을 실행합니다.

```powershell
npm run dev
```

아래처럼 표시되면 성공입니다.

```text
Local: http://localhost:5173/
```

브라우저에서 `http://localhost:5173/`를 열면 후원 룰렛 화면이 나옵니다.

### 다음부터 실행하는 방법

처음 설치가 끝난 뒤에는 아래 한 줄이면 됩니다.

```powershell
npm run dev
```

> `npm install`과 `npm run dev`를 한 번에 붙여 넣지 말고, 반드시 설치가 끝난 뒤 따로 실행하세요.

## 4. 방송 시작 전 설정 방법

웹페이지 위에서부터 순서대로 입력합니다.

### 4-1. 방송 설정

1. 플랫폼 선택: `치지직` 또는 `숲`
2. 채널 ID 또는 방송 주소 입력
   - 치지직은 방송 URL 전체를 붙여 넣어도 됩니다.
3. 표시 이름 입력: 선택 사항
4. 대표 이미지 URL 입력: 선택 사항
5. `1표당 금액` 입력
   - 예: `1000`
6. **방송 설정 적용** 버튼 클릭

### 4-2. 표 계산 방식

후원 금액을 1표당 금액으로 나눈 뒤 소수점은 버립니다.

```text
표 수 = 후원 금액 ÷ 1표당 금액
```

예시:

| 1표당 금액 | 후원 금액 | 추가 표 |
| ---: | ---: | ---: |
| 1,000원 | 5,000원 | 5표 |
| 1,000원 | 2,900원 | 2표 |
| 1,000원 | 500원 | 0표 |

### 4-3. SSAPI 후원 읽기 시작

1. **SSAPI API 키**에 발급받은 키를 입력합니다.
2. 내 개인 PC라면 `이 기기에 저장`을 체크해도 됩니다.
3. **후원 읽기 시작** 버튼을 누릅니다.
4. 상태가 **후원 읽기 중**으로 바뀌면 준비 완료입니다.

버튼을 다시 누르면 후원 읽기가 중단됩니다.

> 후원 읽기 버튼을 누른 이후에 들어온 후원부터 룰렛에 반영됩니다.

---

## 5. 후원 메시지가 룰렛에 들어가는 방식

후원 메시지가 곧 룰렛 선택지 이름입니다.

```text
후원 메시지: 김치찌개
후원 금액: 5,000원
1표당 금액: 1,000원
결과: 김치찌개 +5표
```

아래처럼 공백·기호만 다른 이름은 하나의 항목으로 합칩니다.

```text
김치찌개
김치 찌개
김치찌개!!
```

예를 들어 기존에 `김치찌개`가 5표 있고 `김치 찌개` 후원이 3표 들어오면, 룰렛에는 `김치찌개 8표`로 표시됩니다.

후원 메시지가 없거나, 후원 금액이 1표당 금액보다 작으면 후원 내역에는 남지만 룰렛 표는 추가되지 않습니다.

---

## 6. 방송 전 수동 테스트

실제 후원 전에 꼭 한 번 테스트해 보세요.

1. 화면 아래 `수동 추가`에서 선택지 이름에 `김치찌개` 입력
2. 표 수에 `5` 입력
3. **표 추가** 클릭
4. 룰렛에 새 칸이 생기는지 확인
5. 오른쪽 실시간 순위에 `김치찌개 5표`가 뜨는지 확인
6. `룰렛 돌리기`를 눌러 동작을 확인

수동 추가한 표도 실제 후원 표와 동일하게 룰렛 확률에 반영됩니다.

---

## 7. 룰렛 진행 방법

1. **룰렛 돌리기** 버튼 클릭
2. 팝업 입력칸에 정확히 `도전!` 입력
3. Enter를 누르거나 시작 버튼 클릭
4. 룰렛이 회전한 뒤 결과가 표시됩니다.

룰렛 칸의 크기와 실제 당첨 확률은 표 수 비율대로 결정됩니다.

예:

```text
김치찌개 10표
된장찌개  5표
육회      5표
```

이 경우 김치찌개는 50%, 된장찌개와 육회는 각각 25% 확률입니다.

룰렛이 회전하는 도중에 들어오는 새 후원은 현재 회차에는 반영하지 않고 다음 회차부터 반영합니다.

## 8. 데이터 저장과 초기화

이 프로그램은 별도 데이터베이스를 사용하지 않습니다.

- 룰렛 항목
- 표 수
- 최근 후원 기록
- 중복 후원 방지 기록
- 마지막 방송 설정

이 모든 정보는 현재 PC의 현재 브라우저에 저장됩니다.

### 다른 PC에서 사용하면?

다른 PC나 다른 브라우저에서는 저장된 룰렛이 자동으로 보이지 않습니다. 필요한 경우 수동으로 다시 설정하거나, 같은 PC·같은 브라우저를 사용하세요.

### 처음부터 다시 시작하려면?

상단의 **현재 채널 데이터 초기화**를 누르면 현재 플랫폼·채널 ID에 저장된 룰렛 항목과 후원 기록이 지워집니다.

## 9. 자주 생기는 문제

### `npm install`에서 오래 멈춘 것 같아요

처음 설치는 시간이 걸릴 수 있습니다. 3~5분 이상 아무 변화가 없으면 `Ctrl + C`로 중단한 뒤 아래를 실행해 보세요.

```powershell
npm cache clean --force
npm install --verbose
```

`npm warn Unknown user config "msvs_version"` 또는 `npm warn Unknown user config "python"` 경고는 보통 설치가 멈춘 원인이 아닙니다.

### 후원 읽기 시작을 눌렀는데 연결 오류가 나요

아래를 확인하세요.

1. SSAPI API 키가 정확한지
2. SSAPI 대시보드에서 API 키가 활성 상태인지
3. 인터넷 연결이 정상인지
4. 회사·학교 네트워크나 보안 프로그램이 웹소켓 연결을 막고 있지 않은지
5. 페이지를 새로고침한 뒤 다시 시도했는지

### 후원 읽기 중인데 후원이 안 들어와요

아래 네 가지가 가장 중요합니다.

1. SSAPI 대시보드에 내 채널을 등록했는지
2. 등록할 때 사용한 API 키와 웹앱에 입력한 API 키가 같은지
3. 플랫폼이 같은지
4. 채널 ID가 정확히 같은지

또한 버튼을 누르기 전에 이미 발생한 후원은 이 프로그램에 반영되지 않습니다.

### 후원은 들어왔는데 표가 안 늘어요

- 후원 메시지가 비어 있으면 항목을 만들 수 없습니다.
- 후원 금액이 1표당 금액보다 작으면 표 수는 0입니다.
- 예: 1표당 1,000원, 후원 500원 → 0표

## 10. GitHub Pages로 배포하기

매번 PowerShell에서 `npm run dev`를 실행하기 싫다면 GitHub Pages에 올려서 웹 주소로 사용할 수 있습니다.

1. GitHub에서 새 저장소를 만듭니다. 예: `stream-roulette`
2. 프로젝트 폴더의 파일 전체를 저장소에 올립니다.
3. GitHub 저장소의 `Settings → Pages`로 이동합니다.
4. `Build and deployment`의 Source를 **GitHub Actions**로 선택합니다.
5. `main` 브랜치에 파일을 올리면 자동 배포가 시작됩니다.
6. Actions가 완료되면 Pages 화면에 사이트 주소가 표시됩니다.

이 프로젝트에는 GitHub Pages 자동 배포용 `.github/workflows/deploy.yml` 파일이 이미 포함되어 있습니다.

- SSAPI API 키를 방송 화면, 스크린샷, GitHub 코드, 채팅에 올리지 마세요.
- 공용 PC에서는 `이 기기에 저장`을 체크하지 마세요.
- API 키를 실수로 공개했다면 SSAPI에서 기존 키를 폐기하고 새 키를 발급받으세요.
- 이 웹앱은 개인 서버로 데이터를 보내지 않습니다. 다만 후원을 받기 위해 SSAPI 소켓 서버에는 API 키로 로그인합니다.

## SSAPI 적용 내용

이 프로젝트에는 아래 SSAPI 흐름이 이미 구현되어 있습니다.

```text
SSAPI Socket 연결
→ API 키로 login
→ donation 이벤트만 수신하도록 setReceiver 설정
→ 1분마다 ping
→ donation 이벤트 수신
→ Snappy 압축 해제
→ 후원 메시지와 금액을 룰렛 표로 반영
```

따라서 사용자는 **SSAPI API 키 발급과 내 채널 등록만 하면** 됩니다. 소켓 코드나 별도 서버 코드를 직접 작성할 필요가 없습니다.
