# 후원 룰렛

치지직 또는 숲 방송에서 **후원 메시지를 룰렛 선택지로 만들고, 후원 금액만큼 표를 자동 누적**하는 정적 웹앱입니다.

- 개인 서버를 직접 운영하지 않습니다.
- 방송인 각자의 브라우저에서 동작합니다.
- 룰렛 항목·표·최근 후원 내역은 그 브라우저에만 저장됩니다.
- 실시간 후원 수신은 **SSAPI Socket API**를 사용합니다.

> 이 버전은 치지직·숲 공식 API를 사용하지 않습니다. 따라서 채널명, 프로필 이미지, 방송 제목은 자동으로 불러오지 않고 직접 입력합니다.

---

## 1. 처음 사용 전 알아둘 점

이 웹앱은 단독으로 후원 정보를 찾는 프로그램이 아닙니다. **SSAPI에 내 방송 채널을 등록한 뒤, 같은 SSAPI API 키를 웹앱에 입력하는 방식**입니다.

즉, 방송 시작 전 최초 한 번만 아래 두 작업을 해야 합니다.

1. SSAPI에서 API 키 발급
2. 그 API 키의 룸에 내 치지직 또는 숲 채널 등록

SSAPI는 Socket.IO 4.x 기반으로 동작하고, 후원 데이터는 Snappy 압축 형식으로 전달됩니다. 이 프로젝트는 소켓 로그인, 후원 이벤트 수신, Snappy 압축 해제를 모두 구현해 두었습니다.

## 3. SSAPI 설정 — 꼭 먼저 하세요

### 3-1. API 키 발급

1. [SSAPI 대시보드](https://dashboard.ssapi.kr)에 접속합니다.
2. 회원가입 또는 로그인을 합니다.
3. API 키를 신청하거나 발급받습니다.
4. 발급된 API 키는 복사해 안전한 곳에 보관합니다.

이 키는 이 웹앱이 SSAPI 소켓에 로그인할 때 사용합니다. GitHub 소스 코드에 적거나 다른 사람에게 공유하지 마세요.

### 3-2. 내 방송 채널 등록

SSAPI 대시보드에서 **스트리머 등록** 기능으로 내 방송 채널을 추가합니다.

- 플랫폼: `치지직` 또는 `숲`
- 스트리머 ID: 해당 플랫폼의 내 채널 ID
- API 키: 위에서 발급받은 같은 키의 룸에 등록

치지직 방송 주소가 아래와 같다면:

```text
https://chzzk.naver.com/live/75cbf189b3bb8f9f687d2aca0d0a382b
```

채널 ID는 다음 값입니다.

```text
75cbf189b3bb8f9f687d2aca0d0a382b
```

> **중요:** SSAPI 대시보드에 등록한 플랫폼과 채널 ID가 웹앱에 입력한 값과 정확히 같아야 합니다. 한 글자라도 다르면 후원이 들어와도 룰렛에 반영되지 않습니다.

SSAPI 공식 문서의 기본 흐름도 동일합니다: API 키 발급 → 스트리머 등록 → Socket API로 후원 데이터 수신.

## 4. 내 PC에서 먼저 실행하기

### 4-1. ZIP 풀기

`stream-roulette.zip`을 원하는 폴더에 압축 해제합니다.

예:

```text
C:\WORKSPACE\stream-roulette
```

### 4-2. PowerShell 열기

압축을 푼 폴더에서 마우스 오른쪽 버튼을 누른 뒤 **터미널에서 열기** 또는 **PowerShell에서 열기**를 선택합니다.

아래 명령을 차례대로 실행합니다.

```powershell
npm install
npm run dev
```

성공하면 아래와 비슷한 주소가 나옵니다.

```text
http://localhost:5173
```

해당 주소를 브라우저에서 엽니다.

> `npm install`은 처음 한 번만 필요합니다. 다음부터는 같은 폴더에서 `npm run dev`만 실행하면 됩니다.

## 5. 방송 전 실제 사용 순서

### 5-1. 방송 설정 적용

웹페이지 상단의 **1. 방송 설정**에서 다음을 입력합니다.

1. 플랫폼 선택: `치지직` 또는 `숲`
2. 채널 ID 또는 방송 URL 입력
   - 치지직은 방송 URL을 그대로 붙여 넣어도 됩니다.
   - URL을 넣으면 마지막 경로 값을 채널 ID로 사용합니다.
3. 표시 이름 입력: 선택 사항
4. 대표 이미지 URL 입력: 선택 사항
5. `1표당 금액` 설정
   - 예: 1,000원
6. **방송 설정 적용** 클릭

표 계산 방식은 아래와 같습니다.

```text
표 수 = 후원 금액 ÷ 1표당 금액
소수점은 버림
```

예시:

| 1표당 금액 | 후원금 | 룰렛에 추가되는 표 |
| ---: | ---: | ---: |
| 1,000원 | 5,000원 | 5표 |
| 1,000원 | 2,900원 | 2표 |
| 1,000원 | 500원 | 0표 |

### 5-2. SSAPI 후원 읽기 시작

**2. SSAPI 후원 수신** 영역에서:

1. SSAPI API 키를 입력합니다.
2. 개인 PC에서만 쓸 경우에만 `이 기기에 저장`을 체크합니다.
   - 이 옵션을 켜면 API 키가 현재 브라우저의 localStorage에 평문으로 저장됩니다.
   - PC방·공용 PC에서는 절대 체크하지 마세요.
3. **후원 읽기 시작** 버튼을 누릅니다.
4. 상태가 `후원 읽기 중`으로 바뀌면 성공입니다.

웹앱은 SSAPI에 아래 순서로 연결됩니다.

```text
SSAPI 소켓 연결
→ API 키 로그인
→ donation 이벤트만 수신하도록 설정
→ 1분마다 ping 전송
→ 후원 데이터 수신 및 Snappy 압축 해제
→ 메시지와 금액을 룰렛 표로 반영
```

후원 읽기를 끄려면 같은 버튼을 다시 누릅니다. 중단된 동안의 후원은 이 앱에 반영되지 않습니다.

### 5-3. 후원 메시지와 룰렛 항목

후원 메시지가 곧 룰렛 선택지 이름입니다.

```text
후원 메시지: 김치찌개
후원 금액: 5,000원
1표당 금액: 1,000원
결과: 김치찌개 +5표
```

아래 표현은 같은 선택지로 자동 합산됩니다.

```text
김치찌개
김치 찌개
김치찌개!!
```

공백, 기호, 대소문자 차이를 제거한 이름을 기준으로 비교합니다. 단, 서로 다른 메뉴를 의도적으로 구분하고 싶을 때는 표기 자체를 다르게 입력해 주세요.

### 5-4. 수동 추가로 먼저 테스트하기

실제 후원 전에 아래 **4. 수동 추가**를 이용해 테스트하세요.

1. 선택지 이름에 `김치찌개` 입력
2. 표 수에 `5` 입력
3. **표 추가** 클릭
4. 오른쪽 실시간 순위와 룰렛 칸이 바뀌는지 확인
5. `룰렛 돌리기`를 눌러 정상 동작 확인

### 5-5. 룰렛 돌리기

1. **룰렛 돌리기** 클릭
2. 팝업에 정확히 `도전!` 입력
3. Enter 또는 시작 버튼 클릭
4. 룰렛이 회전한 뒤 당첨 선택지가 표시됩니다.

룰렛이 이미 회전 중일 때 새로 들어온 후원은 이번 회차 결과를 바꾸지 않고 **다음 회차에 반영**됩니다.

룰렛 칸의 크기와 실제 당첨 확률은 표 수 비율을 그대로 사용합니다.

## 6. GitHub Pages에 배포하기

방송용 PC에서 매번 `npm run dev`를 실행하기 싫다면 GitHub Pages에 올려 두세요. 개인 서버를 운영하지 않아도 됩니다.

### 6-1. GitHub 저장소 만들기

1. GitHub에서 새 저장소를 만듭니다.
   - 예: `stream-roulette`
   - 공개 저장소를 권장합니다.
2. 이 프로젝트의 파일 전체를 저장소에 올립니다.
   - `.github/workflows/deploy.yml` 파일도 반드시 포함되어야 합니다.

### 6-2. 파일 올리기 — Git 명령 사용 예시

프로젝트 폴더에서 아래 명령을 실행합니다. `YOUR_GITHUB_ID`와 저장소 이름은 본인 것으로 바꿉니다.

```powershell
git init
git add .
git commit -m "Initial stream roulette"
git branch -M main
git remote add origin https://github.com/YOUR_GITHUB_ID/stream-roulette.git
git push -u origin main
```

GitHub Desktop을 사용해도 됩니다. 이 경우 프로젝트 폴더를 GitHub Desktop에 추가한 뒤, `Publish repository`를 누르면 됩니다.

### 6-3. GitHub Pages 켜기

GitHub 저장소에서:

1. **Settings** 탭을 엽니다.
2. 왼쪽 메뉴에서 **Pages**를 선택합니다.
3. `Build and deployment`의 `Source`를 **GitHub Actions**로 선택합니다.
4. 저장소의 **Actions** 탭을 열어 배포가 끝날 때까지 기다립니다.
5. 완료된 뒤 Pages 화면에 표시되는 사이트 주소를 엽니다.

주소는 보통 아래 형식입니다.

```text
https://GITHUB_ID.github.io/stream-roulette/
```

이 프로젝트에는 GitHub Pages용 자동 배포 파일이 이미 들어 있습니다. `main` 브랜치에 변경 사항을 올릴 때마다 자동으로 다시 배포됩니다.

## 7. 다른 방송인에게 배포할 때

같은 GitHub Pages 주소를 여러 방송인이 사용해도 됩니다. 각 방송인은 아래 정보만 자기 것으로 입력하면 됩니다.

- 자신의 플랫폼
- 자신의 채널 ID
- 자신의 SSAPI API 키
- 자신의 SSAPI 대시보드 스트리머 등록
- 자신의 1표당 금액

룰렛 항목·표·최근 후원 로그는 각 사용자의 브라우저에 따로 저장되므로 자동으로 섞이지 않습니다.

단, 같은 PC의 같은 브라우저에서 같은 플랫폼과 같은 채널 ID를 입력하면 이전에 저장된 룰렛 상태가 다시 나타납니다.

## 8. 자주 생기는 문제

### `후원 읽기 시작`을 눌렀는데 연결 오류가 납니다

다음을 순서대로 확인합니다.

1. 인터넷이 연결되어 있는지 확인
2. SSAPI API 키가 정확한지 확인
3. SSAPI 대시보드에서 API 키가 활성 상태인지 확인
4. 회사망·학교망·보안 프로그램이 WebSocket 연결을 차단하지 않는지 확인
5. 페이지를 새로고침한 뒤 다시 시도

### 상태는 `후원 읽기 중`인데 후원이 안 들어옵니다

다음을 확인합니다.

1. SSAPI 대시보드에 채널을 실제로 등록했는지
2. 등록한 API 키와 웹앱에 입력한 API 키가 같은지
3. 플랫폼이 같은지
4. 채널 ID가 정확히 같은지
5. 후원 읽기 시작 버튼을 누른 **이후에 발생한 후원**인지

### 후원은 들어왔는데 표가 늘지 않습니다

- 후원 메시지가 비어 있으면 선택지를 만들 수 없습니다.
- 후원금이 `1표당 금액`보다 작으면 표 수가 0이 됩니다.
- 예: 1표당 1,000원인데 500원 후원 → 0표

### 다른 PC에서 이전 룰렛 항목이 보이지 않습니다

정상입니다. 이 버전은 서버 DB를 사용하지 않고 현재 브라우저의 localStorage에만 저장합니다. 다른 PC와 자동 동기화되지 않습니다.

### 저장된 룰렛을 처음부터 다시 만들고 싶습니다

상단의 **현재 채널 데이터 초기화**를 누릅니다. 해당 플랫폼/채널의 룰렛 항목, 최근 로그, 중복 처리 기록이 삭제됩니다.

## 9. 보안 주의 사항

- API 키는 깃허브 코드, 방송 화면, 채팅, 스크린샷에 노출하지 마세요.
- 웹앱은 API 키를 별도 개인 서버에 보내지 않습니다. 다만 SSAPI에 로그인하기 위해 브라우저에서 SSAPI 소켓 서버로 전송합니다.
- 공용 PC에서는 `이 기기에 저장`을 체크하지 마세요.
- API 키를 실수로 공개했다면 SSAPI 대시보드에서 폐기하고 새 키를 발급받으세요.

## 구현된 SSAPI 흐름

이 프로젝트는 SSAPI 공식 Socket API의 권장 구조를 따릅니다.

- Socket.IO 4.x 호환 클라이언트로 `https://socket.ssapi.kr`에 연결
- `login` 이벤트로 API 키 전송
- `setReceiver("donation")`으로 후원 이벤트만 선택 수신
- 1분 간격으로 `ping` 전송
- `donation` 이벤트를 Snappy 압축 해제 후 JSON으로 처리
- `_id`를 기준으로 같은 후원을 중복 반영하지 않음

SSAPI 문서:

- [빠른 시작](https://ssapi.kr/docs/quick_start_guide)
- [Socket API 시작하기](https://ssapi.kr/docs/socket/start)
- [후원 donation 이벤트](https://ssapi.kr/docs/socket/donation)
